## 📌 Keep (계속 유지할 것들)
> 잘 되고 있는 것, 계속 유지해야 할 좋은 점들을 기록합니다.

- **일일 스탠드업 미팅**: 매일 오전 10시 15분 스탠드업이 팀원 간 정보 공유에 매우 효과적이었음
- **코드 리뷰 프로세스**: 모든 PR에 대해 최소 1명의 리뷰를 받는 프로세스가 버그 발견에 큰 도움이 됨
- **문서화 습관**: 주요 기능 개발 시 README와 API 문서를 함께 작성한 것이 좋았음
- **Git 브랜치 전략**: feature/기능명 형식의 브랜치 네이밍이 명확하고 관리하기 쉬웠음

## ⚠️ Problem (문제점)
> 개선이 필요한 문제점이나 아쉬운 점들을 기록합니다.

- **커뮤니케이션 부족**: 백엔드와 프론트엔드 간 API 스펙 변경 시 즉시 공유하지 않아 작업 중복 발생
- **배포 프로세스**: 수동 배포로 인해 배포 시간이 오래 걸리고 실수 가능성 존재
- **테스트 커버리지**: 단위 테스트 작성이 일관되지 않아 일부 모듈은 테스트가 부족함
- **문서 접근성**: 문서가 여러 곳에 분산되어 있어 필요한 정보를 찾기 어려움
